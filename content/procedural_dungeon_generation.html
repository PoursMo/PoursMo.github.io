--- layout: default title: Procedural Dungeon Generation ---

When I decided I wanted this game to be a Roguelite, one of the first thing I wanted was to implement a procedural dungeon generation algorithm.
My goal was to generate a set of randomly chosen rooms from an array of room prefabs and interconnect them in such a way that every room is reachable, but there are only certain paths available to use
After doing some research, I liked the approach used for the game "TinyKeep", described in a 10 year old reddit post.
I used the steps outlined here as a guide, adapting them to the vision I had in mind for my game.

Step 1 : Generating rooms

First, I instantiate the room prefabs. Each room is created on a random position inside a circle.

Step 2 : Separation Steering Behavior

I now separate the rooms using Craig Reynolds' separation steering behavior for autonomous characters until there is no overlap.
I added a "space between rooms" parameter to change the overlap size of the rooms to space them out.

Step 3 : Delaunay Triangulation

We now want to create a graph that connects all the rooms together without intersecting lines. This is done using Delaunay Triangulation.
I created 3 structs :
- An "Edge" struct made of 2 vertices.
- A "Triangle" struct made of 3 vertices and 3 edges.
- A "Circle" struct simply made of a center (Vector3) and a radius value. This struct is used to store the circumcircle of the triangles when doing the triangulation.
The triangulation is done by adding points (in our case, rooms) one by one to the triangulation, checking if it is in the circumcircle of any triangle in the triangulation. If it is, we remove those triangles and create new ones with the added point.

Step 4 : Spanning Tree

Now, all the rooms are connected to their neighbors, which results in an uninteresting dungeon with too many paths. So I created a minimal spanning tree from the graph obtained with the triangulation.
The spanning tree will make it so that there is always only 1 possible path from one room to another. A spanning tree is a graph that will always include all of the rooms, making them all reachable.
I created a "Graph" struct to store vertices and edges.
Using Prim's algorithm, I built this tree one vertex at a time, starting with the first vertex (room), at each step adding the cheapest possible connection from the tree to another vertex in the triangulated graph.

Step 5 : Adding Cycles

The tree contains no cycles, which makes the dungeon not very interesting. So I loop through all the edges from the triangulated graph, and if it's not part of the tree I give it a 15% chance of being re-incorporated.
